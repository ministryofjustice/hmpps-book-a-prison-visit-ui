Content split for translations:

* common: service name, global buttons, shared labels, generic hints, back links
* validation: reusable form validation messages, field-level generic errors
* errors: API/business errors and general problem states (not field validation)
* feature namespaces: all page copy specific journeys of areas of functionality. For example, (could change):
  * staticPages - static page content including cookie page
  * bookVisit - book a visit journey
  * etc
